# AGENTS.md

## Commands
- Use npm; this repo has `package-lock.json` and no pnpm/yarn/bun lockfile.
- `npm run dev` starts the Next dev server.
- `npm run build` is the best full verification step; it builds and typechecks the App Router pages.
- `npm exec tsc -- --noEmit` is the focused typecheck command.
- Do not rely on `npm run lint` yet: it runs `next lint` and currently prompts to create ESLint config because no ESLint config exists.

## Dependency State
- `package.json` declares `next@16.2.6`, but `package-lock.json` and installed `node_modules` resolve `next@15.1.6`; `npm ls next` reports this as invalid. Avoid dependency churn unless the task is explicitly to fix/update deps.

## App Structure
- This is a Next App Router site rooted at `app/`; there is no `src/` or `pages/` tree.
- Root layout `app/layout.tsx` fetches GitHub stars server-side via `getStars()` and mounts the global `TopBar` plus client `CommandPalette`.
- `/docs` redirects to `/docs/getting-started`; concrete docs pages are generated by `app/docs/[slug]/page.tsx` from the `ORDER`/`PAGES` registry in `lib/docs.ts`.

## Docs And Search
- Most prose docs are raw HTML strings in `lib/docs.ts`; adding/removing a docs page means updating `PAGES`/`EYEBROWS` and the `RAW` entry together.
- `lib/docs.ts` injects `id`s into `h2`/`h3` headings and `CommandPalette` indexes those headings, so heading text changes also change hash links and search targets.
- The CLI reference is not parsed from the Go project; it is manually mirrored in `lib/cli.ts` and rendered by `components/CliReference.tsx`.
- Changelog content is fetched from `https://raw.githubusercontent.com/Omotolani98/rekord/main/CHANGELOG.md`; `GITHUB_TOKEN` is optional and only used to raise GitHub API limits.

## Styling
- Styling is global in `app/globals.css`; there is no Tailwind, CSS modules, or component-level stylesheet setup.
- The theme is controlled by `data-theme` on `<html>` and localStorage key `rekord-theme`; keep new colors wired through existing CSS variables where practical.
